## 1. Purpose

This file maps the known Control Tower sources, durable technical specifications, active workbook PDFs, missing expected PDFs, and expected future outputs for `docvert`.

Artifact rule: distinguish **project-state files**, **source workbook PDFs**, **technical specifications**, and **generated conversion outputs**. Do not blur those categories.

## 2. Artifact categories

| Category | Meaning | Handling rule |
|---|---|---|
| Durable Control Tower source | Project-governance file for state, decisions, backlog, source status, and phase gates. | Treat as canonical unless superseded in the decision log. |
| Durable pipeline/spec source | Stable technical doctrine, schemas, validation rules, renderer specs, or baseline knowledge. | Use as implementation input and acceptance criteria. |
| Reference-only handoff file | Useful prior context but not automatically canonical alone. | Consult when needed; resolve conflicts through baseline and decision log. |
| Source workbook PDF | Original educational PDF to be converted. | Required for conversion; active only if uploaded/confirmed in the current Project. |
| Generated conversion artifact | Output from a specific conversion run. | Do not treat as source truth. May be used only for comparison. |
| Expected future artifact | Required output contract for future runs. | Must be generated by an authorized implementation/conversion phase. |
| Obsolete/superseded artifact | Based on earlier unsafe or superseded assumptions. | Do not use as governing direction. |
